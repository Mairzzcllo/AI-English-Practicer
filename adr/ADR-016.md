# ADR-016: Conversation Intelligence Layer

## Date
2026-05-13

## Status
Accepted

## Context
The Persona Engine (ADR-013) provides emotional state, relationship tracking, episodic memory, and behavioral policies, but the AI still defaults to "English teacher analysis mode" — over-interpreting every user input with unsolicited teaching and excessive verbosity. Short, natural inputs like "please" or "giving us the news" trigger multi-paragraph lexical analysis instead of natural conversational continuation. The core problem is not persona but **Conversation Intelligence**: the system lacks intent awareness, response constraints, and conversational restraint.

## Decision
Add a Conversation Intelligence Layer with three subsystems, integrated into the existing cognitive pipeline:

### 1. Intent Resolution Layer (Hybrid Cognitive Router)
Three-tier intent classification before response generation:
- **Fast Heuristics** — keyword/pattern-based rules for short/common inputs (e.g., "please" → `continue_conversation`)
- **LLM Semantic Intent** — constrained-output LLM classification as fallback for ambiguous/complex inputs
- **Conversation State** — turn count, short reply streaks, topic changes inform intent disambiguation

Each intent carries a `confidence` score. Low confidence → default to low-interpretation (minimal response).

```
type UserIntent =
  | "continue_conversation"
  | "ask_definition"
  | "ask_correction"
  | "small_talk"
  | "hesitation"
  | "confirmation"
  | "emotional_expression"
```

### 2. Response Compression Policy
Structured budget that constrains generation output:
- `maxSentences` — upper bound on response length
- `explanationDepth` — 0 (none) to 1 (full teach)

Base budget derived from intent; modulated by persona engine (engagement, familiarity, emotional state). Short inputs (<20 chars) → max 1-2 sentences by default.

### 3. Conversational Minimalism & Ambiguity Tolerance
- **Default: do NOT explain/teach** unless intent is explicitly `ask_definition` or `ask_correction`
- **Low-confidence or short input** → natural carry-on (e.g., "Sure — go on"), never lexical analysis
- Teaching is a deliberate mode, not the default

### Pipeline Change
```
input → Intent Resolution → Response Budget → Persona Engine → Prompt Assembly (with constraints)
```

## Consequences
- Eliminates over-interpretation bias for short/common inputs
- Response length proportional to input signal (conversational proportionality)
- Teaching mode only activates when user explicitly signals learning intent
- Persona engine still modulates within budget (engagement boosts enthusiasm, not verbosity)
- LLM semantic classifier in P1; P0 ships with heuristics + state only
