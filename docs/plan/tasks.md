# 任务分解：AI English Mock Interview

## 阶段 1：项目初始化与基础设施

| # | 任务 | 描述 | 预估工时 | 依赖 |
|---|------|------|----------|------|
| 1.1 | 初始化 Next.js 项目 | 使用 create-next-app 创建项目，配置 TypeScript、Tailwind CSS | 0.5天 | - |
| 1.2 | 配置 MongoDB 连接 | 安装 mongoose，建立数据库连接模块 | 0.5天 | - |
| 1.3 | 配置 ESLint + Prettier | 代码风格和 lint 配置 | 0.5天 | 1.1 |
| 1.4 | 项目目录结构和基础布局 | 创建 pages、components、lib、models 等目录，基础 Layout 组件 | 0.5天 | 1.1 |
| 1.5 | 配置环境变量模板 | .env.local.example，配置 AI API Key 等 | 0.5天 | 1.1 |

## 阶段 2：数据模型与 API

| # | 任务 | 描述 | 预估工时 | 依赖 |
|---|------|------|----------|------|
| 2.1 | 定义 Mongoose 数据模型 | Interview Session、Question 数据模型 | 0.5天 | 1.2 |
| 2.2 | 面试流程 API | start / answer / next-question / end 四个接口 | 1天 | 2.1 |
| 2.3 | 历史记录 API | 获取列表和详情 | 0.5天 | 2.1 |
| 2.4 | API 错误处理中间件 | 统一的错误处理和数据验证 | 0.5天 | 2.2 |

## 阶段 3：AI 适配层

| # | 任务 | 描述 | 预估工时 | 依赖 |
|---|------|------|----------|------|
| 3.1 | AI Adapter 接口定义 | 定义 generateQuestion / generateFeedback / generateSummary 接口 | 0.5天 | - |
| 3.2 | OpenAI Adapter 实现 | 接入 OpenAI API，实现适配器 | 1天 | 3.1 |
| 3.3 | DeepSeek Adapter 实现 | 接入 DeepSeek API，实现适配器 | 1天 | 3.1 |
| 3.4 | 多模型配置切换 | 根据环境变量切换模型提供商 | 0.5天 | 3.2, 3.3 |
| 3.5 | Prompt 工程 | 设计各场景（行业/难度）的 Prompt 模板 | 1天 | 3.1 |

## 阶段 4：核心面试 UI

| # | 任务 | 描述 | 预估工时 | 依赖 |
|---|------|------|----------|------|
| 4.1 | 面试首页（行业/难度选择） | 选择面试配置的入口页面 | 0.5天 | 1.4 |
| 4.2 | 面试进行页 UI | 问题展示、回答区、状态指示 | 1天 | 4.1 |
| 4.3 | 语音录制组件 | 基于 Web Speech API 的录音/停止/状态组件 | 1天 | 4.2 |
| 4.4 | 实时语音转文字显示 | 语音识别结果实时展示在回答区 | 1天 | 4.3 |
| 4.5 | TTS 语音播报组件 | 使用 SpeechSynthesis API 播报问题 | 0.5天 | 4.2 |
| 4.6 | 反馈结果展示卡片 | 语法纠错、用词建议、评分的 UI 组件 | 0.5天 | 4.2 |
| 4.7 | 面试总结页面 | 面试结束后的综合评分和建议展示 | 0.5天 | 4.6 |

1. **阶段 1**（项目初始化）→ **阶段 3 的 3.1**（AI 接口定义）
2. **阶段 2**（API + 数据模型）→ 可与阶段 1 并行
3. **阶段 3**（AI 适配层，3.5 Prompt 可提前开始）
4. **阶段 4**（面试 UI）→ 核心交互
5. **阶段 5**（历史记录）
6. **阶段 6**（多行业场景）
7. **阶段 7**（打磨发布）
