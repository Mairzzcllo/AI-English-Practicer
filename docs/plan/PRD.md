# PRD: AI English Mock Interview

## 背景与目标

### 业务背景
英语爱好者需要持续练习口语以保持语感，但找真人练习受限于时间、地域和成本。AI 模拟面试提供了一种随时随地、低成本、无压力的英语练习方式。

### 要解决的问题
- 英语爱好者缺乏日常口语练习场景
- 找真人模拟面试成本高、组织难
- 传统英语学习 App 缺乏真实的面试互动体验
- 用户难以获得即时、针对性的口语反馈

### 成功指标
- MVP 阶段：核心面试流程（AI 提问 → 用户回答 → 反馈）可跑通
- 用户可完成至少 3 种不同行业的模拟面试
- 每次面试后可查看历史记录和反馈

## 范围

### In scope (MVP)
- AI 面试官语音提问（TTS）
- 用户语音回答 + 浏览器 Web Speech API 语音识别
- AI 对回答进行语法/用词纠错和实时文字反馈
- 发音评分
- 多行业面试场景（技术、市场、管理等）
- 面试记录与回放（本地/轻量存储）
- 多模型适配架构（OpenAI / DeepSeek 等）

### Out of scope
- 用户认证/登录系统（MVP 不做）
- 多人协作/分享功能
- 移动端原生 App
- 视频面试

### Future
- OAuth 社交登录
- 面试结果分享
- 发音波形可视化
- 个性化学习计划推荐
- AI 生成针对性练习题

## 用户故事

- As a 英语爱好者, I want to 选择不同行业的面试场景, so that 我能针对性练习我需要的面试类型
- As a 英语爱好者, I want to AI 面试官用英语提问, so that 我能在真实的面试语境中练习
- As a 英语爱好者, I want to 用语音回答面试问题, so that 我既能练习口语也能练习听力
- As a 英语爱好者, I want to 在回答后立即获得语法/用词反馈, so that 我能知道哪里需要改进
- As a 英语爱好者, I want to 获得发音评分, so that 我了解我的发音水平
- As a 英语爱好者, I want to 查看历史面试记录, so that 我能回顾之前的练习并看到进步

## 验收标准

1. Given 用户进入面试页面 / When 选择行业和难度 / Then 系统加载对应的面试题集
2. Given 面试开始 / When AI 提出一个问题 / Then 问题以文字和语音两种形式呈现
3. Given AI 提问完毕 / When 用户开始说话 / Then 系统实时将语音转为文字并显示
4. Given 用户回答完毕 / When 提交回答 / Then AI 返回语法纠错、用词建议和发音评分
5. Given 面试结束 / When 用户查看结果 / Then 展示本次面试的完整记录和综合评分
6. Given 用户有历史记录 / When 进入历史页面 / Then 展示所有历史面试列表，可点开查看详情
